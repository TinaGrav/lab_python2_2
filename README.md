# Алгоритм работы
Функция check_source(source) - проверка источника задач на соответствие контракту. Принимает объект-источник, проверяет через isinstance() соответствует ли он протоколу TaskSource. Если соответствует — вызывает get_tasks(), выводит количество и список задач, иначе выводит сообщение об ошибке

Функция file_input(file_name): принимает имя файла, открывает его в режиме чтения, загружает содержимое JSON файла и возвращает список задач

Класс File_source:  __init__(self, input_file: str = "input_file.json") — инициализация с указанием файла (по умолчанию используется файл "input_file.json", но можно ввести свой. Для этого его необходио сначала добавить в директорию проекта);  get_tasks(self) — возвращает список задач из файла. Если файл не найден, возвращает строку с ошибкой

Функция generate_tasks(file): Принимает имя файла, генерирует случайное количество задач (от 1 до 10), затем для каждой задачи генерирует id и выбирает случайное описание из заранее созданного файла, затем возвращает список сгенерированных задач

Класс Generator_source: __init__(self, descriptions_file: str = "describ_list.txt") — инициализация с указанием файла для генерации описаний задач, get_tasks(self) — вызывает generate_tasks() и возвращает список задач

Класс Input_source: __init__(self) — инициализация, создаёт пустой список задач, get_tasks(self) — запрашивает у пользователя количество задач, затем для каждой задачи запрашивает id и payload. Возвращает список задач. При запросе id и payload используюся функции check_num(num), check_id(id), check_payload(payload), проверяющие введенные данные на корректность
